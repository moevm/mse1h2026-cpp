# Как устроен код

Запускаем через точку входа `main.py`, передавая аргументы через CLI.

## Режимы работы
2 основных режима: инициализация (`init`) и проверка (`check`).

## Структура задания
Для каждого задания/лабы работа ведется в отдельной папке внутри `src/c_course`. Создается три файла:
- типовой `__init__.py`
- класс CLI для инициализации требуемых аргументов
- класс task от `BaseTaskClass`, который содержит набор методов для генерации задания/тестов/проверок (подробнее далее)

Существует отдельная папка для утилит (`base_module`) и аналогично папка с рандомными выражениями для создания случайных строк/массивов/того что нам понадобится. Код в точке входа подтягивает все данные из `src/c_course` по названию папок.

## Передача аргументов
Первый аргумент является позиционным и определяет задание. От задания зависят остальные аргументы, они задаются через флаги. Можно вызвать справку `--help`, чтобы узнать их. Например, `lab3_condition --help` выведет аргументы для задания 3.

Часть аргументов являются **общими для всех работ**:
- `mode` (генерация условия или проверка решения)
- `seed`
- путь до решения
- количество тестов

Т.е. флаг `--seed` есть во всех реализованных задачах.

Библиотека и `main.py` автоматически подтягивают новые работы, нужно только их правильно расположить и правильно назвать.

### Пощелкать
чтобы посмотреть на то как работает код в действии - установить виртуальное окружение и введите из директории workshop следующие команды
```bash
python main.py lab1_hello --help
python main.py lab1_hello --mode init --seed 12 --name Петя
python main.py lab1_hello --mode check --seed 12 --solution "./solution.s"
```

## Создание новой лабораторной работы

### 1. Создание папки
Сделать в папке `src/c_course` папку по шаблону `lab<N>_<codename>`, где `<N>` — номер работы, `<codename>` — кодовое имя. Дальше работа ведётся только в этой папке.

### 2. Создание файла с заданием
Сделать файл с заданием, например, `lab10_test.py`. В этом файле создать класс `Lab10Test`, который наследуется от `BaseTaskClass`.

**Какие методы нужно переопределить:**

- `__init__`: добавить своих аргументов, добавить инициализацию внутренних параметров, добавить в словарь `check_files` вспомогательные файлы для проверки (формат: ключ — имя файла, значение — содержимое файла)

- `generate_task`: добавить вызов генерации задания (seed можно получить через `self.seed`) и вернуть строку с текстовым описанием задания

- `_generate_tests`: добавить генерацию тестов. Тесты нужно создавать в виде класса `TestItem` и добавлять их в список тестов `self.tests`

- `check_sol_prereq`: можно переопределить проверку текста предоставленного задания (наличие `ret`, метки `solution`, ...). Можно оставить базовое

- `compile`: если необходимо поменять опции компиляции, то нужно переопределить этот метод и подставить в вызов `_compile_internal` нужные аргументы. Без острой необходимости переопределять метод `_compile_internal` не рекомендуется

- `run_solution`: аналогично `compile`, только для запуска скомпилированного решения

### 3. Создание CLI-файла
Сделать файл с аргументами CLI: `lab10_cli.py`. Определить две функции:

1. `add_cli_args_lab<N>` — принимает парсер от `argparse` и добавляет аргументы
2. `create_task_lab<N>` — создаёт объект класса задания из переданных аргументов CLI

В добавлении аргументов обязательно выставить функцию обработки как создание задания: 
```python
parser.set_defaults(func=create_task_lab<N>)
```
Создать парсер CLIParser, в котором указать функцию для аргументов и название.

Важно: чтобы появились общие аргументы (--seed и прочие), необходимо вызвать функцию add_common_cli_args.

Важно 2: чтобы одинаково передавать аргументы для базового класса, нужно использовать **get_common_cli_args(args) при создании объекта задания.

### 4. Создание __init__.py
Сделать файл __init__.py, в котором импортировать класс с заданием (Lab10Test) и объект парсера (Lab10CLIParser), чтобы они были сразу доступны на уровне модуля:

python
from .lab10_test import Lab10Test
from .lab10_cli import lab10_parser
### 5. Проверка
После этого работа должна автоматически появиться в библиотеке и main.py. Для примера можно смотреть на base/lab1_hello.
