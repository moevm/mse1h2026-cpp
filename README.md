# mse-template
## О проекте
### Постановка задачи
    Разработать модули и лабораторные работы для введения в C++ и для тестирования на языке C по предмету программирование.

### Требования
    - уникальность заданий: обязательно для лабораторных работ и желательно для заданий модуля, где это возможно; разным людям должны выдаваться разные задания, и каждый студент получает свой опыт работы с новым модулем, не имея возможности успешно запустить код другого студента у себя;
    - задания придумываются нами по найденным нами же источникам; теоретическая часть будет создаваться итеративно, при согласованиях с заказчиком;
    - в работе над проектом подразумевается создание отдельной системы для курса: генерация заданий и проверка решений осуществляется с ее помощью;
    - создание Dockerfile для возможности проверки системы локально;
    - унифицированный формат вывода тестов для студентов: такой, чтобы они представляли подробную информацию о тесте, и такой, чтобы избежать конфликтов с проверяющей системой;
    - важно формулировать все задания так, чтобы избежать «дурацких» вопросов, чтобы студенту было однозначно ясно, что делать, из текста условия;
    - теоретическое наполнение для курсов должно быть, но в виде ссылки на документацию/главу учебника/прочий внешний источник;
### Сценарии использования

### Инструкция по запуску
- сборка
```bash
docker build -t mse-cpp .    
``` 
```bash
docker run -it mse-cpp 
```

- запуск
Из дирертории ./workshop

получение задания
```bash
python3 main.py lab1_hello --mode init --seed 12 --name "student"
```
проверка решения
```bash
python3 main.py lab1_hello --mode check --seed 12 --solution "./solution.c" --name "student"
```
В данный момент, в прототипе, значения для флагов менять нельзя.
