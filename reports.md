# mse-template
## О проекте
### Постановка задачи
    Разработать модули и лабораторные работы для введения в C++ и для тестирования на языке C по предмету программирование.

### Требования
    - уникальность заданий: обязательно для лабораторных работ и желательно для заданий модуля, где это возможно; разным людям должны выдаваться разные задания, и каждый студент получает свой опыт работы с новым модулем, не имея возможности успешно запустить код другого студента у себя;
    - задания придумываются нами по найденным нами же источникам; теоретическая часть будет создаваться итеративно, при согласованиях с заказчиком;
    - в работе над проектом подразумевается создание отдельной системы для курса: генерация заданий и проверка решений осуществляется с ее помощью;
    - создание Dockerfile для возможности проверки системы локально;
    - унифицированный формат вывода тестов для студентов: такой, чтобы они представляли подробную информацию о тесте, и такой, чтобы избежать конфликтов с проверяющей системой;
    - важно формулировать все задания так, чтобы избежать «дурацких» вопросов, чтобы студенту было однозначно ясно, что делать, из текста условия;
    - теоретическое наполнение для курсов должно быть, но в виде ссылки на документацию/главу учебника/прочий внешний источник;

            5. Выполняет инструкцию по запуску генерации условия

    Сценарий 2: локальный запуск с проверкой студенческой работы
        Действующее лицо: заказчик
        Предусловие: у действующего лица уже размещен на хосте прект со всеми его файлами и директориями, открыта командная строка
            1. Переходит в исходную директорию проекта mse-cpp

            3. Выполняет инструкцию по запуску контейнера

            5. Добавляет файл решения
            6. Выполняет инструкцию по запуску проверки работ

    Сценарий 3: локальный запуск с добавлением новых заданий в систему
        Действующее лицо: заказчик/разработчик

            1. Переходит в исходную директорию проекта mse-cpp

            3. Выполняет указанные действия: создает директорю, файлы и переопределяет классы, импортирует нужный модуль

    Сценарий 5: использование в LMS с проверкой студенческой работы
        Действующее лицо: преподаватель

            2. Находит работу конкретного студента
            3. Пользователь видит результат автоматической проверки в системе
            4. Пользователю ничего не нужно делать для проверки задания

    Сценарий 6: использование LMS для добавления нового задания
        Действующее лицо: преподаватель/разработчик/заказчик
        Предусловие: у действующего лица есть нужный доступ к системе LMS

            1. Пользователь заходит в LMS
            2. Пользователь создает в LMS раздел для нового задания
            3. Пользователь открывает это нужное задание
            4. Вставляет кодовый шаблон, предоставленный в исходном коде mse-cpp
            5. Пользователю не нужно писать полноценный код для каждого задания

    Сценарий 7: решение студентом задания на LMS
        Действующее лицо: студент
        Предусловие: у действующего лица есть нужный доступ к системе LMS

            2. Пользователь находит нужное задание
            3. Пользователь открывает нужное задание
            4. Пользователь получает уникальное, сгенерированное для его учетной записи задание
            5. Пользователь решает задание
            6. 
                6.1.1. Вариант: решает верно
                6.1.2. получает однозначное подтверждение верного ответа
                6.2.1. Вариант: решает неверно
                6.2.2. Получает отчет об ошибке
                6.2.3. Понимает из него, в чем дело, т.к. он подробный
                6.2.4. Возврат к п.5


### Инструкция по запуску
- сборка
```bash
docker build -t lab-checker .    
``` 

- запуск

получение справки
```bash
docker run --rm lab-checker --help

```

либо 
```bash 

docker run --rm lab-checker <название задачи> --help

```

получение задания
```bash
docker run --rm lab-checker <название задачи> --mode init

```
проверка решения
```bash
docker run --rm -v "<полный путь к папке с решением>/:/work" lab-checker <название задачи> --mode check --seed 12 --solution "<название файла с решением>"

```
