## Шифрование и дешифрование

Вам предстоит протестировать модуль, реализующий шифрование и дешифрование данных. Модуль содержит два компонента: шифратор и дешифратор, которые работают с текстовыми строками на основе заданного ключа. Шифрование — это преобразование исходного текста таким образом, чтобы его нельзя было прочитать без знания ключа. Дешифрование — это обратное преобразование зашифрованного текста в исходный с использованием того же ключа. Модуль поддерживает два алгоритма шифрования: шифр Цезаря (каждый символ сдвигается на определённое число позиций в алфавите) и XOR-шифрование (каждый символ преобразуется с помощью побитовой операции с ключом). Вы должны провести тестирование "серого ящика" (gray box), что означает, что вам известна внутренняя структура: модуль состоит из двух отдельных функций, которые могут использовать общие вспомогательные функции для обработки ключа. Ваша задача — разработать набор тестов, проверяющих корректность работы модуля на двух уровнях. Не забывайте о том, что важно тестировать граничные случаи.

Вам предоставлены следующие функции (реализация скрыта, но известна спецификация):

1.  `char* encrypt(const char* text, int key)` — функция шифрования. Принимает исходный текст и ключ шифрования. Возвращает новую строку с зашифрованным текстом (память выделяется внутри функции). Возвращает `NULL` при ошибке (например, если передан пустой текст или неверный ключ).
2.  `char* decrypt(const char* cipher, int key)` — функция дешифрования. Принимает зашифрованный текст и ключ. Возвращает новую строку с расшифрованным текстом (память выделяется внутри функции). Возвращает `NULL` при ошибке.
3.  `int get_last_error()` — вспомогательная функция, возвращающая код последней ошибки:
    *   `0` — ошибок нет
    *   `1` — неверный ключ (выходит за допустимый диапазон)
    *   `2` — пустая строка
    *   `3` — неизвестный алгоритм
4.  `void set_algorithm(int algo)` — функция выбора алгоритма шифрования. Вызывается один раз перед началом работы. Возможные значения:
    *   `1` — Цезарь (сдвиг символов)
    *   `2` — XOR (побитовая операция)


#### 1. Модульное тестирование (Unit Tests)

Проверьте изолированно каждую функцию модуля. Так как это "серый ящик" (gray box), Вы можете вызывать их напрямую и проверять результаты.

*   **Тестирование выбора алгоритма (`set_algorithm`):**
    *   Проверьте установку алгоритма Цезаря (запустите алгоритм шифрования и убедитесь, что шифрование происходит сдвигом кода символа на указанное число)
    *   Проверьте установку алгоритма XOR (запустите алгоритм шифрования и убедитесь, что шифрование происходит XOR'ом с ключом)
    *   Проверьте обработку некорректного номера алгоритма

*   **Тестирование шифрования (`encrypt`) для алгоритма Цезаря:**
    *   Проверьте шифрование строки из одного символа с ключом 1
    *   Проверьте шифрование строки с буквами и пробелами
    *   Проверьте шифрование с ключом 0 (строка не должна измениться)
    *   Проверьте обработку пустой строки 

*   **Тестирование шифрования (`encrypt`) для алгоритма XOR:**
    *   Проверьте шифрование строки из одного символа с ключом 0xFF
    *   Проверьте шифрование длинной строки (убедитесь, что все символы преобразуются)
    *   Проверьте, что повторное применение XOR с тем же ключом возвращает исходный текст

*   **Тестирование дешифрования (`decrypt`):**
    *   Для алгоритма Цезаря: зашифруйте известную строку с ключом K, затем расшифруйте результат и проверьте, что получилась исходная строка
    *   Для алгоритма XOR: аналогичная проверка
    *   Проверьте дешифрование с неправильным ключом (результат должен отличаться от исходного текста)

#### Вариативность (параметры варианта задания определяются по хешу от ФИО)

Ниже приведены параметры, которые зависят от варианта.  


1.  **Алгоритм шифрования по умолчанию (если не вызван `set_algorithm`):**
    *   Цезарь
    *   XOR

2.  **Чувствительность к регистру для шифра Цезаря:**
    *   да (заглавные и строчные буквы сдвигаются отдельно, сохраняя регистр)
    *   нет (все буквы приводятся к одному регистру перед сдвигом)
    *   частичная (заглавные сдвигаются в диапазоне заглавных, строчные — в диапазоне строчных)

3.  **Множество символов, доступных для шифрования:**
    *   только латинские буквы (A-Z, a-z)
    *   латинские буквы и цифры
    *   расширенная ASCII (коды 0-255)

4.  **Обработка символов вне допустимого множества:**
    *   остаются без изменений
    *   удаляются из строки
    *   заменяются на специальный символ-заполнитель
