## Шифрование и дешифрование (Цезарь / XOR) 

Вам предстоит протестировать модуль, реализующий шифрование и дешифрование данных. Модуль содержит два компонента: шифратор (encrypt) и дешифратор (decrypt), которые работают с символьными строками на основе заданного ключа. Поддерживаются два алгоритма: шифр Цезаря (сдвиг символов) и XOR-шифрование (побитовая операция с ключом). Вы должны провести тестирование "серого ящика" (gray box), что означает, что вам известна внутренняя структура: модуль состоит из двух отдельных функций, которые могут использовать общие вспомогательные функции для обработки ключа. Ваша задача — разработать набор тестов, проверяющих корректность работы модуля на двух уровнях. Не забывайте о том, что важно тестировать граничные случаи.


Вам предоставлены следующие функции (реализация скрыта, но известна спецификация):

1.  `char* encrypt(const char* text, int key)` — функция шифрования. Принимает исходный текст и ключ шифрования. Возвращает новую строку с зашифрованным текстом (память выделяется внутри функции). Возвращает NULL при ошибке 
2.  `char* decrypt(const char* cipher, int key)` — функция дешифрования. Принимает зашифрованный текст и ключ. Возвращает новую строку с расшифрованным текстом (память выделяется внутри функции). Возвращает NULL при ошибке
3.  `int get_last_error()` — вспомогательная функция, возвращающая код последней ошибки (0 — ошибок нет, 1 — неверный ключ, 2 — пустая строка, 3 — неизвестный алгоритм)
4.  `void set_algorithm(int algo)` — функция выбора алгоритма шифрования (1 — Цезарь, 2 — XOR). Вызывается один раз перед началом работы


#### 1. Модульное тестирование (Unit Tests)

Проверьте изолированно каждую функцию модуля. Так как это "серый ящик" (gray box), Вы можете вызывать их напрямую и проверять результаты.

*   **Тестирование выбора алгоритма (`set_algorithm`):**
    *   Проверьте установку алгоритма Цезаря (алгоритм 1)
    *   Проверьте установку алгоритма XOR (алгоритм 2)
    *   Проверьте обработку некорректного номера алгоритма

*   **Тестирование шифрования (`encrypt`) для алгоритма Цезаря:**
    *   Проверьте шифрование строки из одного символа с ключом 1
    *   Проверьте шифрование строки с буквами и пробелами
    *   Проверьте шифрование с ключом 0 (строка не должна измениться)
    *   Проверьте обработку пустой строки (должен вернуться NULL и установиться код ошибки)

*   **Тестирование шифрования (`encrypt`) для алгоритма XOR:**
    *   Проверьте шифрование строки из одного символа с ключом 0xFF
    *   Проверьте шифрование длинной строки (убедитесь, что все символы преобразуются)
    *   Проверьте, что повторное применение XOR с тем же ключом возвращает исходный текст

*   **Тестирование дешифрования (`decrypt`):**
    *   Для алгоритма Цезаря: зашифруйте известную строку с ключом K, затем расшифруйте результат и проверьте, что получилась исходная строка
    *   Для алгоритма XOR: аналогичная проверка
    *   Проверьте дешифрование с неправильным ключом (результат должен отличаться от исходного текста)

#### 2. Интеграционное тестирование (Integration Tests)

Проверьте корректность взаимодействия шифратора и дешифратора при последовательной работе (в пайплайне).

*   Создайте тесты для указанного в варианте алгоритма с разными ключами (1, 5, -3, 26). Для каждого ключа: зашифруйте исходную строку, затем расшифруйте результат и проверьте, что полученная строка совпадает с исходной
*   Проверьте, что функция `get_last_error` корректно возвращает код ошибки, если в одном из этапов произошёл сбой 

#### 3. Сквозное тестирование (End-to-End Tests)

Проверьте полный цикл работы с файлами, имитируя реальное использование системы шифрования.

*   Создайте временный текстовый файл `input.txt` с некоторым содержимым (несколько строк текста).
*   Прочитайте содержимое файла в строку, зашифруйте её с выбранным алгоритмом и ключом, сохраните результат в новый файл `encrypted.txt`.
*   Прочитайте файл `encrypted.txt`, расшифруйте его содержимое с тем же ключом, сохраните результат в файл `decrypted.txt`.
*   Сравните содержимое исходного файла `input.txt` и расшифрованного файла `decrypted.txt` — они должны быть идентичны.
*   Подумайте над граничными случаями. Вот некоторые из возможных: пустой файл, файл с симовлами разных языков, очень длинный файл.
