
# Задание:

Вам нужно реализовать шаблонный класс `Matrix3x3`, который представляет матрицу фиксированного размера 3×3. Класс должен иметь общую реализацию для типов int, bool и double, но специализированное поведение для логических матриц (`bool`) и матриц с плавающей точкой (`double`).

**Класс должен включать:**
- `typename T` - тип элементов матрицы (всегда размер 3×3)

**Методы:**

**1) Конструкторы**
- `Matrix3x3()` - конструктор по умолчанию, создает единичную матрицу (1 на главной диагонали, 0 в остальных ячейках)
- `Matrix3x3(const T& value)` - заполняет всю матрицу указанным значением
- `Matrix3x3(const T& a11, const T& a12, const T& a13, const T& a21, const T& a22, const T& a23, const T& a31, const T& a32, const T& a33)` - создает матрицу из 9 элементов (по строкам)

**2) Доступ к элементам**
- `T& at(size_t i, size_t j)` - доступ к элементу с проверкой границ
    - Если индексы выходят за пределы (не 0, 1 или 2), выбрасывает исключение `std::out_of_range`
- `T at(size_t i, size_t j) const` - доступ к элементу только для чтения

**3) Операции**
- `Matrix3x3 add(const Matrix3x3& other) const` - сложение матриц (поэлементное)
- `Matrix3x3 multiply(const Matrix3x3& other) const` - умножение матриц
- `void fill(const T& value)` - заполняет матрицу значением
- `bool isIdentity() const` - проверяет, является ли матрица единичной

**Специализация 1: `Matrix3x3<bool>`**
Для логических матриц изменить поведение следующих методов:
- `add()` - использовать логическое **OR** вместо обычного сложения
- `multiply()` - использовать логическое **AND** вместо умножения и **OR** вместо сложения

**Специализация 2: `Matrix3x3<double>`**
Для матриц с плавающей точкой добавить:
- `Matrix3x3<int>` round() const` - возвращает новую матрицу с типом `int`, где каждый элемент округлен до ближайшего целого
