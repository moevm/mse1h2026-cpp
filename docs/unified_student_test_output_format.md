# Унифицированный формат вывода студенческих тестов (USTOF)

**Версия:** 1.0\
**Автор:** Белаид Фарук - 3342\
**Дата:** 2026-02-22

------------------------------------------------------------------------

## 1. Назначение

Данный документ описывает **унифицированный формат вывода результатов
студенческих тестов**, реализованных на языке программирования C.

Формат предназначен для автоматического сравнения:

-   результатов тестирования студента;
-   результатов эталонной проверяющей системы.

Вывод должен быть детерминированным, машинно‑читаемым и пригодным для
автоматизированной обработки и сравнения результатов тестирования.

------------------------------------------------------------------------

## 2. Общие требования

1.  Вывод осуществляется в **stdout**.
2.  Кодировка --- UTF‑8.
3.  Каждая запись занимает ровно одну строку.
4.  Каждая строка оканчивается символом новой строки (`\n`).
5.  Порядок вывода строго фиксирован.

Допустимые разделы:

-   RUN
-   ROUND
-   CASE
-   SUMMARY
-   RESULT

Любой отладочный или произвольный текст запрещён.

------------------------------------------------------------------------

## 3. Структура вывода

Вывод состоит из следующих записей (в строгом порядке):

RUN → ROUND → CASE → SUMMARY → RESULT

------------------------------------------------------------------------

## 4. Заголовок RUN

Выводится один раз в начале.

Формат:

RUN task=`<task_id>`{=html} student=`<student_hash>`{=html}
seed=`<seed>`{=html}

Пример:

RUN task=sum student=8f3a12c0 seed=42

Поля:

-   task --- идентификатор задания.
-   student --- hash, полученный из ФИО студента.
-   seed --- базовое значение генератора случайных чисел.

------------------------------------------------------------------------

## 5. Запись ROUND

Выводится перед тестовыми случаями раунда.

Формат:

ROUND id=`<round_id>`{=html} params=`<parameter_list>`{=html}

Пример:

ROUND id=0 params=k=7,m=3

Параметры должны детерминированно вычисляться из:

(student_hash, seed, round_id)

------------------------------------------------------------------------

## 6. Запись CASE

Выводится для каждого тестового случая.

Формат:

CASE r=`<round_id>`{=html} i=`<case_id>`{=html}
input=`<serialized_input>`{=html} expected=`<value>`{=html}
got=`<value>`{=html} ok=\<0\|1\>

Поля:

-   r --- номер раунда.
-   i --- номер тестового случая.
-   input --- сериализованные входные данные.
-   expected --- ожидаемый результат.
-   got --- полученный результат.
-   ok --- результат проверки (1 --- успешно, 0 --- ошибка).

Пример:

CASE r=0 i=3 input=a=5,b=7 expected=12 got=10 ok=0

------------------------------------------------------------------------

## 7. Правила сериализации данных

### Целые числа

x=5

### Вещественные числа

value=3.14

### Строки

s="hello world"

Кавычки обязательны.

### Массивы

arr=\[1,2,3\]

### Составные структуры

p=(x=1,y=2)

------------------------------------------------------------------------

## 8. Запись SUMMARY

Выводится один раз после выполнения всех тестов.

Формат:

SUMMARY pass=`<count>`{=html} fail=`<count>`{=html}

Пример:

SUMMARY pass=95 fail=5

------------------------------------------------------------------------

## 9. Запись RESULT

Выводится последней строкой.

Допустимые значения:

RESULT OK

или

RESULT FAIL

OK --- результаты тестирования студента совпадают с эталонной системой.

FAIL --- обнаружены расхождения.

------------------------------------------------------------------------

## 10. Требования к детерминизму

Реализация должна гарантировать:

-   одинаковый seed → одинаковые тестовые данные;
-   одинаковый hash студента → одинаковые параметры;
-   одинаковый запуск → идентичный вывод.

------------------------------------------------------------------------

## 11. Пример полного вывода

RUN task=sum student=8f3a12c0 seed=42 ROUND id=0 params=k=7 CASE r=0 i=0
input=a=1,b=2 expected=3 got=3 ok=1 CASE r=0 i=1 input=a=10,b=10
expected=20 got=19 ok=0 SUMMARY pass=1 fail=1 RESULT FAIL

------------------------------------------------------------------------

Будущие расширения формата должны сохранять обратную совместимость.
