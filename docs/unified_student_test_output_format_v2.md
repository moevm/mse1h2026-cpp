## 1. Назначение

Данный документ описывает унифицированный формат вывода результатов
выполнения студенческих тестов в лабораторной работе по дисциплине
«Программирование».

Формат используется:

-   студентом --- для вывода результатов разработанных им тестов;
-   проверяющей системой курса --- для автоматической проверки
    корректности тестирования.

Проверяющая система сравнивает вывод программы студента с эталонным
выводом.

------------------------------------------------------------------------

## 2. Общие требования

1.  Вывод осуществляется в **stdout**.
2.  Кодировка --- UTF‑8.
3.  Каждая запись выводится **строго одной строкой**.
4.  Каждая строка заканчивается символом новой строки (`\n`).
5.  Порядок записей фиксирован.

Типы записей:

-   RUN
-   CASE
-   SUMMARY
-   RESULT

Дополнительный отладочный вывод запрещён.

------------------------------------------------------------------------

## 3. Структура вывода

Вывод должен иметь следующий порядок:

RUN → CASE → SUMMARY → RESULT

------------------------------------------------------------------------

## 4. Запись RUN

Выводится один раз в начале программы.

Формат:

RUN task=`<task_id>`{=html}

где:

-   task --- идентификатор задания.

Пример:

RUN task=sum

------------------------------------------------------------------------

## 5. Запись CASE

Каждый тест выводится отдельной строкой.

Формат:

CASE id=`<case_id>`{=html} input=`<данные>`{=html}
expected=`<value>`{=html} got=`<value>`{=html} ok=\<0\|1\>

где:

-   id --- номер теста;
-   input --- входные данные теста в текстовом виде;
-   expected --- правильный результат работы функции (по эталонной
    реализации, предоставленной преподавателем);
-   got --- результат работы тестируемой реализации;
-   ok --- результат проверки:
    -   1 --- тест пройден;
    -   0 --- тест не пройден.

Пример:

CASE id=0 input=a=5,b=7 expected=12 got=12 ok=1

------------------------------------------------------------------------

## 6. Представление входных данных

Входные данные указываются в текстовом виде.

Рекомендуемые примеры записи:

Целые числа:

x=5

Строки:

s="hello"

Массивы:

arr=\[1,2,3\]

Составные значения:

p=(x=1,y=2)

Конкретный формат входных данных определяется заданием.

------------------------------------------------------------------------

## 7. Запись SUMMARY

Выводится после всех тестов.

Формат:

SUMMARY pass=`<count>`{=html} fail=`<count>`{=html}

где:

-   pass --- количество успешно пройденных тестов;
-   fail --- количество непройденных тестов.

Пример:

SUMMARY pass=9 fail=1

------------------------------------------------------------------------

## 8. Запись RESULT

Выводится последней строкой.

Допустимые значения:

RESULT OK

или

RESULT FAIL

OK --- вывод студента совпадает с эталонным выводом проверяющей системы.

FAIL --- обнаружены расхождения.

------------------------------------------------------------------------

## 9. Пример полного вывода

RUN task=sum CASE id=0 input=a=1,b=2 expected=3 got=3 ok=1 CASE id=1
input=a=10,b=10 expected=20 got=19 ok=0 SUMMARY pass=1 fail=1 RESULT
FAIL
