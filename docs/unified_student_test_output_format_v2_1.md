## 1. Назначение

Данный документ описывает унифицированный формат вывода результатов
выполнения студенческих тестов в лабораторной работе по дисциплине
«Программирование».

Формат используется:

-   студентом --- для вывода результатов разработанных им тестов;
-   проверяющей системой курса --- для автоматической проверки
    корректности тестирования.

Чтобы снизить вероятность ошибок при ручном формировании строк вывода,
формирование записей должно выполняться с использованием заранее 
предоставленных функций библиотеки курса. Студент передаёт в эти функции
только значения параметров теста.

Пример рекомендуемых сигнатур функций:

``` c
void print_run(const char *task_id);

void print_case(
    int case_id,
    const char *input,
    const char *expected,
    const char *got,
    int ok
);

void print_summary(int pass, int fail);

void print_result(int success);
```

Проверяющая система сравнивает вывод программы студента с эталонным
выводом.

------------------------------------------------------------------------

## 2. Общие требования

1.  Вывод осуществляется в `stdout`.
2.  Кодировка --- UTF-8.
3.  Каждая запись выводится строго одной строкой.
4.  Каждая строка заканчивается символом новой строки `\n`.
5.  Порядок записей фиксирован.

Типы записей:

-   RUN
-   CASE
-   SUMMARY
-   RESULT

Дополнительный отладочный вывод запрещён.

------------------------------------------------------------------------

## 3. Структура вывода

RUN → CASE → SUMMARY → RESULT

------------------------------------------------------------------------

## 4. Запись RUN

Формат:

    RUN task=<task_id>

Пример:

    RUN task=sum

------------------------------------------------------------------------

## 5. Запись CASE

Формат:

    CASE id=<case_id> input=<данные> expected=<value> got=<value> ok=<0|1>

где:

-   `id` --- номер теста;
-   `input` --- входные данные теста в текстовом виде;
-   `expected` --- правильный результат работы функции (по эталонной
    реализации, предоставленной преподавателем);
-   `got` --- результат работы тестируемой реализации;
-   `ok` --- результат проверки:
    -   `1` --- тест пройден;
    -   `0` --- тест не пройден.

Пример:

    CASE id=0 input=a=5,b=7 expected=12 got=12 ok=1

------------------------------------------------------------------------

## 6. Представление входных данных

Общие правила:

-   используются латинские символы нижнего регистра;
-   пробелы внутри записи не допускаются;
-   пары `имя=значение` разделяются запятой без пробелов;
-   порядок параметров соответствует порядку аргументов функции.

### Целые числа

    x=5

### Строки

    s="hello"

Строки обязательно заключаются в двойные кавычки.

### Массивы

    arr=[1,2,3]

Элементы массива разделяются запятой без пробелов.

### Составные значения (структура)

Составное значение представляет собой набор именованных полей (например,
структуру).

    point=(x=1,y=2)

где:

-   имя структуры записывается слева от знака равенства;
-   поля перечисляются внутри круглых скобок;
-   поля разделяются запятой без пробелов.

Конкретный набор параметров определяется заданием.

------------------------------------------------------------------------

## 7. Запись SUMMARY

Формат:

    SUMMARY pass=<count> fail=<count>

где:

-   `pass` --- количество успешно пройденных тестов;
-   `fail` --- количество непройденных тестов.

Пример:

    SUMMARY pass=9 fail=1

------------------------------------------------------------------------

## 8. Запись RESULT

Допустимые значения:

    RESULT OK

или

    RESULT FAIL

OK — вывод студента совпадает с эталонным выводом проверяющей
системы.\
FAIL --- обнаружены расхождения.

------------------------------------------------------------------------

## 9. Пример полного вывода

    RUN task=sum
    CASE id=0 input=a=1,b=2 expected=3 got=3 ok=1
    CASE id=1 input=a=10,b=10 expected=20 got=19 ok=0
    SUMMARY pass=1 fail=1
    RESULT FAIL
