Идея 1: Система управления складом (Warehouse Management System)
Суть проекта: Разработать систему учета товаров на складе с поддержкой разных типов хранения (температурный режим, хрупкость) и стратегий обработки заказов.

Структура и требования:

1. ООП:
Базовый класс Item.
Классы-наследники: Electronics, Food, Furniture и т.д.

2. Шаблоны:
Шаблонный класс Storage<RackType, Capacity> — стеллаж, рассчитанный на хранение только определенного типа товаров.
Внутри Storage содержит контейнер (например, std::array фиксированного размера или std::vector с максимальной емкостью) для хранения товаров.
Методы: addItem(std::unique_ptr<RackType> item), removeItem(int index), getItem(int index).
Параметр RackType — конкретный тип товара (Electronics, Food...), который можно хранить в этом стеллаже.

3. Управление ресурсами (RAII, умные указатели):
Класс Warehouse владеет стеллажами через std::vector<std::unique_ptr<StorageBase>>. Но так как Storage шаблонный, нужен общий базовый класс (StorageBase) с виртуальными методами для полиморфизма (см. пункт "Стирание типа").
Стеллаж (Storage) владеет товарами через std::vector<std::unique_ptr<RackType>>.
std::shared_ptr<Item> теперь используется, когда товар изымается со стеллажа для отгрузки или инвентаризации и передается в другой отдел. Стеллаж при этом временно "теряет" владение.

4. STL и алгоритмы:
Контейнеры:
std::map<std::string, std::vector<StorageInfo>> для быстрого поиска, в каком стеллаже лежит товар по категории.
std::vector<std::unique_ptr<StorageBase>> для списка всех стеллажей на складе.
Алгоритмы:
Поиск стеллажа с наименьшей загрузкой (std::min_element + лямбда).
Поиск товара по названию по всем стеллажам (std::find_if для каждого стеллажа).
Подсчет общего объема товаров (std::accumulate по всем стеллажам).

5. Перегрузка операторов:
Перегрузить << для вывода информации о товаре (без изменений).
Перегрузить << для вывода информации о стеллаже (тип, загрузка, вместимость).
Перегрузить < или > для сравнения стеллажей по заполненности.

6. Исключения:
OutOfSpaceException — при попытке добавить товар в заполненный стеллаж.
TypeMismatchException — при попытке добавить Food в стеллаж для Electronics (бросается в методе Storage::addItem).
TemperatureBreachException — если товар Food с отрицательной температурой пытаются положить в обычный стеллаж (логика в Warehouse).


Генерация вариантов (по хешу):
Набор классов: Какие 3 конкретных типа товаров реализовывать (например, из списка [Electronics, Food, Furniture, Chemicals, Clothes]).
Тип сортировки: По возрастанию или убыванию цены/объема.
Операторы: Перегрузить == или !=.
Тип контейнера: Для корзины заказа использовать std::vector или std::list.
