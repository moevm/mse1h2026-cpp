Идея 2: Шифровальщик файлов/строк (Encryption Utility)
Суть проекта: Программа для шифрования/дешифрования данных с использованием различных алгоритмов.

Структура и требования:

1. ООП:
Абстрактный класс Cipher с методом process(const std::string& data).
Классы: CaesarCipher, XorCipher, ReverseCipher (наследники).

2. Шаблоны:
Шаблонный класс SecureBuffer<T>, где T — тип данных (char, byte). Он будет хранить зашифрованные данные и автоматически их затирать в деструкторе.

3. Управление ресурсами (RAII):
SecureBuffer в деструкторе заполняет выделенную память нулями (memset), а потом освобождает.
Использовать std::unique_ptr<Cipher> для выбора алгоритма.

4. STL и алгоритмы + Лямбды:
Работа со строками.
Для шифрования Цезаря использовать std::transform и лямбду, которая сдвигает символ.
Контейнеры: std::vector<char> как буфер.

5. Перегрузка операторов:
Перегрузить operator^ для класса XorCipher (или для SecureBuffer), который будет применять шифрование XOR.
Перегрузить operator<< для вывода зашифрованных данных (например, в hex-формате).

6. Исключения:
EncryptionException базовое.
InvalidKeyException (если ключ нулевой длины для XOR).

Генерация вариантов (по хешу):
Набор шифров: Реализовать 3 конкретных алгоритма из списка [Caesar, Xor, Reverse, Atbash, Vigenere].
Режим: Только шифрование или шифрование+дешифрование.
Тип ключа: Целочисленный сдвиг или строковый ключ.
Исключения: Какое исключение бросать при ошибке валидации ключа.
