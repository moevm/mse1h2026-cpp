Идея №2: Библиотека математических выражений (Math Expression Library)
Суть проекта: Создать парсер и вычислитель математических выражений с поддержкой переменных и различных функций.

Структура и требования:

1. ООП (Компоновщик):
Абстрактный базовый класс ExpressionNode с методом evaluate().
Классы-наследники: NumberNode (константа), VariableNode (переменная), BinaryOperationNode (+, -, *, /), FunctionNode (sin, cos).

2. Шаблоны:
Шаблонный класс Calculator<T>, где T — тип вычислений (int, double, Complex из следующего пункта). Методы: parse(), solve().

3. Управление ресурсами (RAII, умные указатели):
Дерево разбора (AST) хранить с помощью std::unique_ptr<ExpressionNode>, чтобы гарантировать удаление всех узлов.
Хранилище переменных (контекст) сделать через std::map<std::string, std::shared_ptr<Value>>, чтобы переменные можно было использовать в нескольких выражениях.

4. Перегрузка операторов:
Обязательно! Создать класс комплексного числа Complex (или рациональной дроби Rational) и перегрузить для него все арифметические операции (+, -, *, /), а также << и >>.

5. STL и алгоритмы:
std::map для хранения переменных и их значений.
std::vector для хранения списка токенов при лексическом анализе.
Алгоритмы для обработки строк (удаление пробелов).

6. Лямбда-выражения:
Для реализации пользовательских функций. Хранить в std::map<std::string, std::function<double(double)>>, куда можно добавлять лямбды.

7. Исключения:
SyntaxErrorException, DivisionByZeroException, UnknownVariableException.

Генерация вариантов (по хешу):
Тип вычислений: int, double или Complex.
Набор операций: Разрешить только сложение/умножение или все 4 операции.
Функции: Какие встроенные функции поддерживать (sin, cos, log, sqrt).
Исключения: В каком случае бросать специфическое исключение (например, при переполнении стека вызовов).
