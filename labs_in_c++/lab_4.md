# Параллельный парсер логов с многопоточностью
Легенда: Есть огромный файл с логами (несколько гигабайт). Нужно быстро подсчитать статистику по уровням логирования (INFO, WARNING, ERROR, DEBUG) и собрать все ERROR сообщения для отдельного отчета.

Что нужно сделать:
Напишите программу, которая:

Принимает путь к файлу и количество потоков для обработки.

Разбивает файл на примерно равные куски (по количеству потоков). Важно разбивать по границам строк, чтобы не разорвать строку лога пополам.

Запускает несколько потоков, каждый обрабатывает свой кусок файла.

Каждый поток считает свою локальную статистику:

Количество строк каждого уровня

Собирает все ERROR строки в свой локальный вектор (или хранит их позиции)

После завершения всех потоков, программа объединяет результаты:

Суммирует статистику по уровням

Объединяет все ERROR сообщения (или выводит их в отдельный файл)

Использовать синхронизацию минимально (каждый поток работает со своими данными, финальное объединение — в главном потоке).

Дополнительные требования:

Обрабатывать ошибки (файл не найден, недостаточно прав).

Корректно завершать потоки при ошибках.

Вывести время выполнения обработки.

Пример структуры лога:

2024-01-15 10:23:45 INFO Application started
2024-01-15 10:23:46 DEBUG Initializing modules
2024-01-15 10:23:47 ERROR Failed to load config: file not found
2024-01-15 10:23:48 WARNING Using default configuration

На что обратить внимание:

Работа с многопоточностью (std::thread)

Разделение файла между потоками без конфликтов

Работа с файловой системой (большие файлы, позиционирование)

Синхронизация (хоть и минимальная)

Объединение результатов из разных потоков

Замер производительности

Потенциально работа с регулярными выражениями для парсинга
