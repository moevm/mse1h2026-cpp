# Лабораторная работа 4: OOП 

OOП в C++ включает классы, наследование, полиморфизм и инкапсуляцию. В C это имитируется структурами и функциями. C++ добавляет виртуальные функции для динамического связывания. 

 

Задания 

N=3: (1) базовый класс (Shape/Animal/Vehicle), (2) производных (2-4), (3) метод (area/sound/speed). Хеш варьирует. 

Пример: базовый=Animal, производных=3, метод=sound. 

 

1: Создайте класс [базовый] с методом [метод] и [производных] наследника с override. 

 

2: Инкапсуляция и наследование с доступом к protected-членам 
Создайте базовый класс [базовый], который: 
Содержит закрытые (private) поля: 
string name — название объекта. 
int id — уникальный идентификатор (можно генерировать автоматически). 
Предоставляет публичные (public) аксессоры (геттеры/сеттеры) для этих полей. 
Содержит защищённое (protected) поле m_data — массив фиксированного размера (например, 10 целых чисел), доступное напрямую только наследникам. 
Объявляет виртуальный метод [метод]() с реализацией по умолчанию, которая выводит информацию о базовом объекте. 
Создайте [производных] класса-наследника, которые: 
Наследуют базовый класс публично. 
В конструкторе инициализируют базовую часть и заполняют защищённый массив m_data своими значениями (например, для первого наследника — чётные числа, для второго — нечётные, и т.д.). 
Переопределяют метод [метод](): 
Сначала вызывают базовую версию метода. 
Затем выводят содержимое своего массива m_data. 
Демонстрируют доступ к закрытым полям базового класса только через аксессоры (прямой доступ запрещён!). 
В функции main: 
Создайте объекты каждого класса-наследника. 
Продемонстрируйте работу методов [метод](). 
Покажите, что через указатель/ссылку на базовый класс вызываются правильные переопределённые методы (полиморфизм). 
Продемонстрируйте работу геттеров/сеттеров для закрытых полей. 


3: Множественное наследование интерфейсов 
Создайте иерархию, используя множественное наследование от двух абстрактных базовых классов (интерфейсов): 
Первый базовый класс: [базовый] с чисто виртуальным методом [метод](). 
Второй базовый класс: Drawable с чисто виртуальным методом draw(). 
Создайте [производных] конкретных класса, которые наследуют оба интерфейса и реализуют все виртуальные методы. 
Каждый класс должен иметь уникальную реализацию [метод]() и draw(). 
В main создайте массив (или вектор) указателей на [базовый] и продемонстрируйте полиморфизм. Также создайте массив указателей на Drawable и покажите, что те же объекты можно использовать через второй интерфейс. 

4: CRTP с иерархией классов 
Реализуйте шаблонный базовый класс [базовый]Base<T>, который использует CRTP для статического полиморфизма: 
Базовый класс содержит метод info(), который выводит название конкретного типа (используя typeid(T).name()) и вызывает реализацию [метод]() через static_cast<T*>(this). 
Создайте [производных] классов-наследников: [базовый]1, [базовый]2, ... (количество равно параметру). 
Каждый наследник реализует свой вариант метода [метод](). 
В main продемонстрируйте: 
Использование каждого класса по отдельности. 
Невозможность хранить разные типы в одном контейнере (как ограничение статического полиморфизма). 
Альтернативу — использование std::variant или шаблонной функции для работы с любым типом.



5: Разрешение ромбовидного наследования 
Создайте иерархию, демонстрирующую проблему ромба (diamond problem): 
Базовый класс [базывый] содержит: 
Приватное поле id (уникальный идентификатор). 
Виртуальный метод [метод]() с реализацией по умолчанию. 
Чисто виртуальный метод type(). 
Создайте два промежуточных класса, наследующих [базывый]: 
Каждый переопределяет [метод]() по-своему. 
Каждый добавляет своё поле. 
Создайте финальный класс, наследующий оба промежуточных. Используйте виртуальное наследование для разрешения неоднозначности. 
В финальном классе: 
Разрешите конфликт наследования [метод]() (явно укажите, какую версию использовать или напишите свою). 
Реализуйте метод type(). 
Добавьте метод showHierarchy(), который через RTTI (typeid, dynamic_cast) определяет и выводит информацию о всех типах в иерархии данного объекта. 
В main создайте объект финального класса и продемонстрируйте: 
Вызов [метод](). 
Вызов showHierarchy(). 
Попытку привести объект к каждому из промежуточных классов (через dynamic_cast) и вызов их методов.
