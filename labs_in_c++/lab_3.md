Идея 3: Менеджер распределенных задач (Distributed Task Manager)
Суть проекта: Система для постановки, выполнения и отслеживания статуса задач в "пуле потоков" (эмулируем).
(Эта идея напрямую перекликается с вашим примером про Task и Manager).

Структура и требования:

1. ООП (Иерархия задач):
Базовый класс Task с виртуальным методом execute().
Классы: CalcTask (вычислить что-то), IOTask (симуляция ввода-вывода), DelayTask (просто ждет).

2. Управление ресурсами (RAII):
Класс TaskManager инкапсулирует std::vector<std::shared_ptr<Task>>. Задачи добавляются через addTask(std::unique_ptr<Task> task), который внутри конвертируется в shared_ptr (так как на задачу могут ссылаться и менеджер, и поток).

3. Шаблоны:
Класс Result<T> для хранения результата выполнения задачи. Если задача еще не выполнена — внутри std::nullopt (C++17) или std::optional.

4. STL и алгоритмы + Лямбды:
Контейнеры: std::deque<std::shared_ptr<Task>> для очереди задач.
Алгоритмы: Удалить выполненные задачи из очереди (std::remove_if + лямбда, проверяющая флаг isDone()).
Сортировка задач по приоритету (std::sort + лямбда, сравнивающая getPriority()).

5. Перегрузка операторов:
Перегрузить () для классов-задач (функторы), чтобы их можно было вызывать как функции.
Перегрузить << для вывода статуса задачи ([Выполнено] Задача ID: 1).

6. Исключения:
TaskFailedException — если задача бросила исключение во время execute(). Менеджер должен поймать это исключение и пометить задачу как "ошибка".

Генерация вариантов (по хешу):
Типы задач: Реализовать 3 конкретных типа из списка: [CalcTask, IOTask, DelayTask, NetworkTask, FileTask].
Контейнер: std::vector или std::deque для очереди.
Сортировка: По приоритету или по времени создания.
