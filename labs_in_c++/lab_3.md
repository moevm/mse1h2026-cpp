#Текстовый редактор с историей изменений (стек откатов)

Легенда: Мы пишем упрощенную версию текстового редактора с поддержкой отмены действий (undo).

Что нужно сделать:
Создайте класс TextEditor, который хранит текущий текст и историю изменений.

Интерфейс класса:

void append(const std::string& text) — добавить текст в конец

void erase(int n) — удалить последние n символов

void undo() — отменить последнее действие (append или erase)

std::string getText() const — получить текущий текст

Требования к реализации:

История изменений должна храниться в виде стека (std::stack или свой стек через std::vector).

Каждое действие должно сохранять состояние, достаточное для отката (можно хранить либо противоположное действие, либо снимки изменений).

Подумайте, что хранить в "командах": можно хранить пару (тип_действия, данные).

Отмена действия (undo) должна восстанавливать состояние ДО выполнения этого действия.

Если нечего отменять, undo() ничего не делает.

Пример работы:

TextEditor editor;
editor.append("Hello");
editor.append(" World");
std::cout << editor.getText(); // "Hello World"
editor.undo();
std::cout << editor.getText(); // "Hello"
editor.erase(3);
std::cout << editor.getText(); // "Hel"
editor.undo();
std::cout << editor.getText(); // "Hello"

На что обратить внимание:

Работа со структурами данных (стек)

Проектирование классов и хранение состояния

Нужно придумать формат хранения истории (паттерн Command)

Управление памятью и копированием строк
