Идея 5: Генератор тестов (Quiz/Test Generator)
Суть проекта: Программа для создания, хранения и проведения тестов. Есть вопросы, варианты ответов, подсчет результатов. 

Структура и требования:

1. ООП (Иерархия вопросов):
Абстрактный базовый класс Question с чисто виртуальными методами:
bool checkAnswer(const std::string& userAnswer) — проверка ответа.
std::string getText() — получить текст вопроса.
int getScore() — получить баллы за вопрос.

Классы-наследники (минимум 3 разных типа):
ChoiceQuestion — вопрос с вариантами ответов (A, B, C, D). Хранит std::map<char, std::string> варианты и правильный ответ (char).
TrueFalseQuestion — вопрос да/нет. Хранит bool правильный ответ.
TextQuestion — вопрос с открытым ответом (строка). Хранит std::string правильный ответ (регистронезависимая проверка).

2. Шаблоны:
Шаблонный класс Statistics<T> для сбора статистики по разным типам данных.
Например, Statistics<int> для подсчета среднего балла, Statistics<double> для времени прохождения, Statistics<std::string> для самых частых ошибок.
Реализовать методы addValue(T value), getAverage(), getMin(), getMax().

3. Управление ресурсами (RAII, умные указатели):
Класс Test (тест) содержит std::vector<std::unique_ptr<Question>>. Вопросы создаются через new и сразу передаются в вектор.
Класс TestSession (сессия прохождения) хранит указатели на вопросы теста и ответы пользователя. Можно использовать std::shared_ptr, если нужно шарить результаты между разными отчетами.
Деструкторы автоматически чистят всю память.

4. STL и алгоритмы + Лямбды:
Контейнеры: std::vector для списка вопросов, std::map<int, std::string> для ответов пользователя (номер вопроса -> ответ), std::set<int> для пропущенных вопросов.

Алгоритмы:
std::shuffle — перемешать вопросы при новом прохождении теста.
std::count_if + лямбда — посчитать количество правильных ответов.
std::max_element + лямбда — найти вопрос с наибольшим количеством ошибок.
std::for_each + лямбда — вывести все вопросы, на которые даны неправильные ответы.

5. Перегрузка операторов:
Перегрузить << для вывода вопроса вместе с вариантами (для ChoiceQuestion).
Перегрузить == для сравнения двух вопросов (по тексту и правильному ответу) — полезно для избегания дубликатов.
Перегрузить += для класса Test, чтобы добавлять вопросы: myTest += std::make_unique<ChoiceQuestion>(...).

6. Исключения:
Создать иерархию исключений для теста:
TestException (базовое)
QuestionNotFoundException (запросили вопрос с несуществующим номером)
InvalidAnswerException (пользователь ввел "F", когда есть только A,B,C,D)
EmptyTestException (попытка начать тест без вопросов)

Генерация вариантов (по хешу от имени студента):
Набор типов вопросов: Каждый студент реализует 3 конкретных класса из списка: [ChoiceQuestion, TrueFalseQuestion, TextQuestion, MathQuestion, MultipleChoiceQuestion, OrderQuestion (расставить по порядку)].
Тип контейнера для ответов: std::map или std::unordered_map.
Тип сортировки: В каком порядке показывать статистику — по алфавиту (по названию вопроса) или по количеству ошибок (убывание/возрастание).
Какие операторы перегрузить: Например, только << или << + == (определяется хешем).
Какое исключение реализовать детально: InvalidAnswerException или QuestionNotFoundException.ц
