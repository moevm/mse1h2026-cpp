# Лабораторная работа 5:  Smart Pointers 

Умные указатели (unique_ptr, shared_ptr) автоматически управляют памятью, используя RAII для избежания утечек. В C память управляется вручную (malloc/free). 

 

Задания 

N=3: (1) тип указателя (unique/shared/weak), (2) ресурс (int/array/object), (3) операция (create/use/reset). Хеш определяет. 

Пример: тип=shared, ресурс=array, операция=use. 

 

1:Базовые операции с умными указателями 
Создайте умный указатель заданного [тип] (unique_ptr, shared_ptr или weak_ptr), управляющий ресурсом [ресурс] (int, массив int[] или объект пользовательского класса Test). 
В зависимости от операции: 
Если операция create — просто создайте указатель и выведите сообщение о создании. 
Если операция use — для unique_ptr и shared_ptr выведите значение ресурса; для weak_ptr — попробуйте получить доступ через lock() и выведите значение или сообщение, что ресурс недоступен. 
Если операция reset — для unique_ptr и shared_ptr сбросьте указатель и проверьте, что ресурс освобождён; для weak_ptr — сбросьте указатель и проверьте expired(). 
Важно: Для weak_ptr операция use и reset возможны только если есть соответствующий shared_ptr. В задаче предполагается, что студент создаёт и shared_ptr для демонстрации 

2: Передача умных указателей в функции 
Реализуйте три функции, демонстрирующие разные способы передачи умных указателей заданного [тип] на ресурс [ресурс]: 
Функция, принимающая указатель по значению — демонстрирует перемещение (для unique_ptr) или копирование (для shared_ptr). Для weak_ptr — принимает по значению и пытается получить доступ через lock(). 
Функция, принимающая указатель по ссылке — позволяет модифицировать оригинальный указатель (например, вызвать reset). 
Функция, принимающая сырой указатель — получает сырой указатель из умного через .get() и работает с ресурсом, не управляя временем жизни. 
В main создайте умный указатель, выполните заданную [операция] (create/use/reset) с его использованием через эти функции, и продемонстрируйте разницу в поведении. 

3: Класс с умным указателем в качестве члена 
Создайте класс ResourceManager, который содержит умный указатель заданного [тип] на ресурс [ресурс]: 
Конструктор класса создаёт ресурс (или принимает уже существующий указатель). 
Реализуйте методы, соответствующие заданной [операция]: 
Если операция create — метод acquire(), который создаёт новый ресурс (если ещё не создан). 
Если операция use — метод use(), который выводит значение ресурса или выполняет действие над ним. 
Если операция reset — метод release(), который освобождает ресурс. 
Для weak_ptr в качестве члена класса: класс также должен содержать shared_ptr (или получать его извне), чтобы weak_ptr мог существовать осмысленно. 
Продемонстрируйте работу класса в main, показывая автоматическое освобождение ресурса при уничтожении объекта ResourceManager. 

4: Циклические ссылки и их разрешение 
Создайте структуру из двух классов, которые ссылаются друг на друга: 
struct Node; // предварительное объявление 

struct Parent { 
   std::vector<[тип]_ptr<Node>> children; 
   ~Parent() { std::cout << "Parent destroyed\n"; } 
   void [операция](); // метод, соответствующий заданной операции 
}; 
 
struct Node { 
   [тип]_ptr<Parent> parent; 
   ~Node() { std::cout << "Node destroyed\n"; } 
   void [операция](); // метод, соответствующий заданной операции 
};
В зависимости от типа указателя: 
Если задан shared_ptr — создайте циклическую ссылку, покажите, что объекты не уничтожаются (утечка памяти), затем исправьте ситуацию, заменив одно из полей на weak_ptr. 
Если задан unique_ptr — такая циклическая ссылка невозможна (компилятор не позволит). Покажите это и объясните, почему unique_ptr не подходит для циклических структур. 
Если задан weak_ptr — продемонстрируйте корректное использование для разрыва циклов в паре с shared_ptr. 
Для операции: 
create — создайте связную структуру из нескольких объектов. 
use — пройдите по структуре и выведите информацию о связях. 
reset — разорвите связи в определённом порядке и покажите, как объекты уничтожаются. 

5: Пользовательский deleter для разных сценариев 
Создайте умный указатель заданного [тип] с пользовательским deleter'ом для ресурса [ресурс]: 
Для ресурса int: deleter должен не освобождать память, а выводить сообщение и обнулять значение (демонстрация нетривиального deleter'а). 
Для ресурса array: deleter должен использовать delete[] и выводить статистику (сколько элементов было в массиве). 
Для ресурса object: deleter должен вызывать специфический метод cleanup() у объекта перед удалением, а также логировать время удаления. 
Дополнительные требования: 
Если тип указателя shared_ptr, покажите, что deleter вызывается ровно один раз при удалении последнего владельца, даже если копировался. 
Если тип указателя unique_ptr, покажите, что deleter является частью типа и разные deleter'ы создают разные типы. 
Если тип указателя weak_ptr — эта задача не имеет смысла, поэтому для этого случая создайте shared_ptr с custom deleter и наблюдайте за ним через weak_ptr. 
Для заданной [операция]: 
create — создайте указатель и покажите, какой deleter установлен. 
use — используйте ресурс, затем принудительно вызовите deleter (через reset() или выход из области видимости). 
reset — сбросьте указатель и убедитесь, что deleter вызван.
